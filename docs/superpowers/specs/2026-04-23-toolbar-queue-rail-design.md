# Toolbar and Queue Rail Design

- 日期：2026-04-23
- 状态：Approved
- 目标：精简右上角全局操作，并将左侧任务队列改为更易扫读的轨道式列表。

## 1. 背景

当前顶部工具栏同时承载保存、启动、运行所选、停止、重试、跳过等按钮，信息密度过高。左侧任务队列以普通卡片列表呈现，任务顺序、状态和当前选中项不够直观。

## 2. 方案

采用保守重排：

1. 顶部工具栏只保留全局操作：
   - `保存队列`
   - 单一运行按钮：空闲时为 `启动队列`，运行时为 `停止队列`
2. 任务级操作下沉到编辑面板右上角：
   - `运行所选任务`
   - `重试任务`
   - `跳过任务`
   - `上移任务`
   - `下移任务`
   - `删除任务`
3. 左侧任务列表改为轨道式：
   - 左侧圆形序号作为任务位置锚点。
   - 任务卡片显示任务名、状态、计划文件数量和工作目录短路径。
   - 当前任务使用更高层级 surface 突出。

## 3. 视觉规范

必须遵守 `DESIGN.md`：

- 使用 Azure Clarity 的浅色 surface token。
- 主按钮使用 `#004ac6 -> #2563eb` 渐变。
- 容器边缘使用 ghost border，不新增强硬分割线。
- 列表项通过 tonal layering、圆角和留白表达层级。

## 4. 验收标准

1. 顶部工具栏不再显示 `运行所选任务`、`重试任务`、`跳过任务`。
2. 顶部工具栏保留 `保存队列` 和动态 `启动队列` / `停止队列`。
3. 任务级操作出现在编辑面板的操作区。
4. 左侧任务列表包含轨道序号元素和短路径展示。
5. 测试、类型检查和构建通过。
